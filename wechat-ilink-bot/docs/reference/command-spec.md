# 命令规范

## 命令格式

本 Bot 支持四类命令前缀 + 视频触发，由 `ModeRouter` 按优先级路由（入口 `RateLimiter` 限流）：

| 前缀 / 触发 | 处理器 | 用途 |
|------|--------|------|
| `#` | FarmMode → GameEngine | 农场游戏命令 |
| `/` | SystemCommandMode | 系统命令（模式切换、会话管理、帮助、状态） |
| `!` | AutogameMode → MCP | autogame 指令（调 autogame-xcx MCP server） |
| 上传视频 | ReviewMode（抢占式） | 视频评测任务 |
| 其它文本 | 当前模式（默认 ChatMode） | LLM 聊天或原样回显 |

### 农场命令格式

```
#<命令名>
#<命令名> <参数>
#<命令名> <参数> x<数量>
```

- 所有游戏指令必须以 `#` 开头（全角 `＃` 自动归一为半角，中文输入法常误输全角）
- `#` 为指令触发前缀，由 ModeRouter 剥离后传给 GameEngine
- 参数以空格分隔；命令尾随中文标点（`。！？，、；：`）与全角空格自动归一/剥离
- 购买数量支持裸数字或 `x` 前缀（如 `#购买 小麦 3` 或 `#购买 小麦 x3`）
- 作物名支持口语别名（`麦`/`萝卜`/`西红柿`/`苞米`/`莓`/`瓜` 等），完整名仍可用
- 支持中文别名；打错字时给出"你是不是想输入 X"近似建议

### 系统命令格式

```
/<命令名>
/<命令名> <参数>
```

- 系统命令以 `/` 开头，由 SystemCommandMode 解析
- 不进入 GameEngine，不影响农场状态

### autogame 命令格式

```
!<命令名>
!<命令名> <参数>
```

- autogame 指令以 `!` 开头，由 AutogameMode 解析，经 MCP 调用 autogame-xcx 远程服务
- 仅在 `autogame-config.json` 启用（`enabled=true`）且 MCP server（默认 `localhost:8765`）连接时生效；否则回复"MCP 服务未启用"

## 系统命令（7 个）

由 `SystemCommandMode` 处理；前 6 个所有用户均可使用，`/sudo` 仅管理员可见可用：

### `/mode`

- **参数**：`[chat|claude|autogame|farm|review]`
- **说明**：查看或切换当前模式
- **示例**：
  - `/mode` — 查看当前模式
  - `/mode chat` — 切换到 LLM 聊天模式（默认）
  - `/mode claude` — 切换到 Claude Bridge 模式（普通文本经 `claude` 子进程处理，支持跨消息会话延续）。**副作用**：当 `claudeBridgeAdminDefaultPrivileged=true` 且 userId 在 `claudeAdminUsers` 白名单内时，进入即默认切提权档（省去 `/sudo on`）；提权 transient，`/sudo off` 或重启回收
  - `/mode autogame` — 切换到 Autogame 模式（`!` 指令经 MCP 驱动 autogame-xcx）
  - `/mode farm` — 回复"通过 # 前缀直接使用"（Farm 模式无需切换，始终可通过 `#` 触发）
  - `/mode review` — 回复"通过上传视频自动触发"（Review 模式为抢占式）
- **持久化**：切换成功后写入 `player.bot_mode` 列，下次启动仍生效
- **输出**（切换到 chat）：
  ```
  已切换到 chat 模式
  ```
- **输出**（切换到 claude）：
  ```
  已切换到 Claude Bridge 模式，发送 /new 开始新会话，或 /sessions 查看历史会话。
  ```

### `/new`

- **参数**：无
- **说明**：清空当前的活跃 Claude 会话（`activeClaudeSessionId`），下一条消息将创建全新会话上下文
- **语义**：仅对 Claude Bridge 模式有意义；活跃会话仅存于内存，重启后自动失效
- **输出**：`已开始新的 Claude 会话。`

### `/sessions`

- **参数**：无
- **说明**：列出当前用户最近的 Claude 会话（按更新时间倒序，最多 10 条）
- **数据来源**：`claude_sessions` 表（`ClaudeSessionRepository.findByUserIdOrderByUpdatedDesc`）
- **输出**：
  ```
  最近的 Claude 会话：
  1. 帮我写一个脚本（06-24 16:54）
  2. (无标题)（06-23 10:12）

  发送 /resume <序号> 恢复对应会话。
  ```

### `/resume`

- **参数**：`<序号|会话ID>`
- **说明**：恢复指定历史 Claude 会话，将其设为活跃会话；若当前不在 claude 模式则一并切换
- **示例**：
  - `/resume 1` — 按 `/sessions` 列表序号恢复
  - `/resume 857e1a35-...` — 按会话 ID 恢复（仅限本人会话）
- **输出**：`已恢复 Claude 会话：<标题>`

### `/help`

- **参数**：无
- **说明**：显示系统命令清单（不含农场命令，农场帮助用 `#帮助`）
- **输出**：列出 `/mode`、`/status`、`/new`、`/sessions`、`/resume`、`/plan`、`/approve`、`/help` 的用法（管理员另显示 `/sudo`）

### `/status`

- **参数**：无
- **说明**：显示当前模式与玩家状态摘要
- **输出**：
  ```
  当前模式：chat
  👤 user123  💰 1200  Lv.5
  ```

### `/sudo`

- **参数**：`[on|off|status]`（缺省等价 `status`）
- **权限**：**仅 `claudeAdminUsers` 白名单内的 userId**；非管理员命中按未知命令处理（不暴露该指令存在），`/help` 也仅对管理员显示该项
- **说明**：切换本会话 Claude Bridge 提权档——`on` 后子进程以 `claudeBridgePrivilegedMode`（`bypassPermissions`）运行，可读写/执行宿主机任意路径；`off` 回到只读工作目录。提权 transient，重启回收。详见 [claude-bridge 设计](../design/claude-bridge.md)
- **示例**：
  - `/sudo on` — 开启无限制模式
  - `/sudo off` — 关闭，回到受限只读
  - `/sudo` / `/sudo status` — 查看当前档

### `/plan`

- **参数**：`[on|off|status]`（缺省等价 `status`）
- **权限**：所有用户（plan 模式只读，与 default 同安全级）
- **说明**：切换 Claude 计划档——`on` 后子进程以 `claudeBridgePlanMode`（`plan`）+ 只读白名单运行，只读探索产出方案文本（不写不改不执行）；`off` 回到默认只读。与 `/sudo` 互斥（`on` 时清 `claudePrivileged`，反之亦然）。transient，重启回收
- **示例**：
  - `/plan on` — 开启计划模式，发消息让 Claude 产出方案
  - `/plan off` — 关闭，回到默认只读
  - `/plan` — 查看当前档

### `/approve`

- **参数**：无
- **权限**：所有用户可批准；**完整执行（写/命令）仅管理员**（自动切 bypass）
- **说明**：批准执行上一轮计划——置位一次性 `claudeApprovedExec` 标志，下一条消息由 `ClaudeBridgeMode` 拼"执行上一轮计划"前缀 + `--resume` 同会话触发；管理员自动提权到 bypass，非管理员保持只读档（写操作被拒并提示）。标志一次性，消费/切会话/重启即清。无活跃会话则提示先用 `/plan on` 产出方案
- **示例**：`/plan on` 出方案 → `/approve` → 发"执行"

## autogame 命令（6 个）

由 `AutogameMode` 处理（`!` 前缀），经 MCP 调用 [wechat-link-autogame-xcx](../../../wechat-link-autogame-xcx) 暴露的 tools。详见 [mcp-autogame 设计](../design/mcp-autogame.md)。MCP 未启用/未连接时统一回复"MCP 服务未启用"。

| 命令 | MCP tool | 说明 |
|------|----------|------|
| `!list` | `list_templates` | 列出可用模板 |
| `!run <名称>` | `run_template`({name}) | 执行模板（**异步**：先回执"开始执行"，完成后回执结果） |
| `!status` | `get_status` | 查询执行状态 |
| `!stop` | `stop_execution` | 请求停止 |
| `!report` | `get_report` | 查看上次执行报告 |
| `!help`（或 `!` 空） | — | 渲染 registry 中的 tool 列表 |

- **示例**：`!run 签到`
- **异步语义**：`!run` 先回执"▶ 开始执行模板：<名称>"，任务在 daemon 单线程池串行执行（与 Python 端串行调度对应），完成后回执"✓ 执行完成：<结果>"或"✗ 执行失败/异常：<msg>"
- **超时/错误**：MCP 调用抛 IOException → 回复"调用失败：<msg>"

## 农场游戏命令（27 个）

### 基础信息

#### `USER_INFO` / `我的信息`
- **参数**：无
- **说明**：查看玩家信息（昵称、金币、等级、经验）。玩家名优先显示昵称（`#改名` 设置），未设置时用 `农夫#<wxid尾4位>` 兜底
- **示例**：`USER_INFO`、`我的信息`
- **输出**：
  ```
  👤 玩家: 大丰收
  💰 金币: 1200
  ⭐ 等级: Lv.5 (320/500 EXP)
  📅 连续签到: 7天
  ```

#### `VIEW_FARM` / `查看农场`
- **参数**：无
- **说明**：查看当前农场所有地块状态。**作物成熟由时间驱动**：读取时按 `plantedAt + growTimeMinutes` 实时推进 stage（`CropGrowth` 读时计算，无定时器）。若有作物已被偷，末尾追加"被偷提醒"
- **示例**：`VIEW_FARM`、`查看农场`
- **输出**：
  ```
  🌾 帮帮农场 - 我的土地
  [01]🌾小麦-生长中  [02]🥕胡萝卜-成熟✓  [03]🟫 空地
  💰 金币: 1200 📦 种子: 5
  🕵️ 提醒：你有 3 个作物已被偷（收获时少收）
  ```

### 批量操作

#### `PLANT_ALL` / `一键种植`
- **参数**：`[作物名]`（可省略，省略时复用最近一次购买/种植的作物）
- **说明**：在所有空地块种植指定作物
- **前置**：有空地块、有足够种子

#### `HARVEST_ALL` / `一键收获`
- **参数**：无
- **说明**：收获所有成熟作物到背包。**被偷扣减**：某地块被偷的产量（`steal_record`）从本次收获中扣除，收获后清理该地偷菜记录
- **错误**：无成熟作物

#### `SELL_ALL` / `一键卖菜`
- **参数**：无
- **说明**：出售背包中所有收获物获得金币
- **错误**：背包中无收获物

#### `CLEAR_ALL` / `一键锄地`
- **参数**：无
- **说明**：清理所有枯萎作物，重置地块为空地。按清理块数写入**除草榜**（`WEED`）
- **别名**：`一键锄地`、`锄地`
- **错误**：无枯萎作物

#### `WATER_ALL` / `一键浇水`
- **参数**：无
- **说明**：给所有有作物的地块浇水。按浇水块数写入**浇水榜**（`WATER`）

#### `PEST_ALL` / `一键除虫`
- **参数**：无
- **说明**：给所有有作物的地块除虫。按除虫块数写入**除虫榜**（`PEST`）

#### `FERTILIZE` / `施肥`
- **参数**：`[作物名]`（可省略，省略时复用最近一次购买/种植的作物）
- **说明**：施肥增加产量（消耗肥料道具）
- **前置**：有作物、有肥料
- **示例**：`#施肥 小麦`、`#施肥`（复用上次作物）

### 昵称

#### `RENAME` / `改名`
- **参数**：`<昵称>`（1–12 字符）
- **说明**：设置农场昵称，榜单与偷菜界面展示用（SDK 不提供微信昵称，需玩家自设）
- **别名**：`改名`、`昵称`、`起名`
- **示例**：`#改名 大丰收`
- **错误**：昵称为空、超过 12 字符

### 社交（偷菜）

#### `STEAL` / `偷菜`
- **参数**：无（列候选）/ `<序号>`（对候选执行偷取）
- **说明**：全服随机池偷菜。**两步交互**（因裸数字回复不走 `#` 路由）：
  - `#偷菜` — 从全服随机挑最多 3 个有成熟作物的邻居列出（带昵称 + 作物），候选缓存 5 分钟
  - `#偷菜 <序号>` — 偷取该邻居价值最高的成熟地块
- **规则**：仅偷成熟作物；每次偷走约 30% 产量（`max(1, round(yield*0.3))`）；同一贼对同一地同一成熟周期只能偷一次（`steal_record` PK）；偷成功后 5 分钟冷却；被偷产量记入 `steal_record`，受害者收获时扣减
- **别名**：`偷菜`、`偷`、`偷菜查询`
- **产出**：偷到的作物入偷菜者背包，并按偷得价值写入**偷菜大盗榜**（`STEAL`）
- **冷启动**：随机池无成熟作物时回复"现在没有可偷的成熟作物，过会儿再来～"

### 排行榜

#### `RANK_MENU` / `排行`

- **说明**：排行榜总入口，列出可查的各类榜单
- **别名**：`排行`、`排行榜`

#### `WEALTH_RANK` / `财富榜`

#### `LEVEL_RANK` / `等级榜`

#### `STEAL_RANK` / `偷菜榜`
- **说明**：偷菜大盗榜，按累计偷得价值（`action_rank` 的 `STEAL`）。别名：`偷菜榜`、`偷菜排行`、`大盗榜`

#### `WEED_RANK` / `除草排行`

#### `WATER_RANK` / `浇水排行`
